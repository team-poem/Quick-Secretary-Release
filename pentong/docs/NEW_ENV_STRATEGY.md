# 새 환경 적응 전략 (2026-04-21~)

**배경**: 개발 PC 에서 잘 돌아가는 뚝딱비서가 새 테스트 PC (다른 Windows 버전·한컴
버전·로캘·AV 설정) 로 옮기면 반복적으로 깨지는 현상. 이걸 일회성 패치로 대응하는
대신 **체계적 방어 축**을 정해 단계별로 쌓아 올린다.

## 4가지 축

### A. Observability — 원격 진단 용이성
**증상**: 새 PC 에서 문제 났을 때 사용자-개발자 왔다 갔다 하면서 cmd 명령, 로그,
버전 정보 하나씩 받는 진단 핑퐁에 수 시간 소요.

**해결**:
- [v0.0.16] SDK connect 3회 실패 시 `~/.ddukddak/sdk_debug.log` 에 환경 정보
  자동 덤프 (Win 버전 / 로캘 / CLI·Node·Python 버전 / `claude -p` 테스트 결과).
  사용자는 이 파일 하나만 공유.
- [향후] 설정 창에 **"진단 정보 복사"** 버튼 — 클립보드에 한 번에 담아 전달.
- [향후] 에러 발생 시 "문제 보고" 다이얼로그 — 로그 요약과 재현 단계 자동
  작성해 메일/메시지로 전송.

### B. Self-healing — 자동 복구
**증상**: 일시적 환경 요인 (AV 첫 스캔, 느린 디스크, 네트워크 지연) 으로 실패
하는 경우가 "안 됩니다" 로 곧장 노출.

**해결**:
- [v0.0.16] SDK connect 실패 시 **자동 재시도 3회** (간격 2초, 5초). 일시적
  지연 흡수.
- [향후] `-p` 비대화형 방식으로 **폴백 동작** — SDK 가 계속 실패해도 옛 방식
  으로라도 작업 수행 가능 (기능 제한 있지만 완전 불능은 아님).
- [향후] `claude auth login` 만료 감지 시 설정 창 자동 오픈 + 로그인 유도.

### C. Pre-flight — 사전 점검
**증상**: 문제가 실제 사용 시점에 터짐. 사용자가 메시지 입력하고 결과 기다리는
중 실패 → 사용자 좌절감 최대.

**해결**:
- [향후] 앱 최초 실행 시 (또는 설정 창 열 때) **환경 일괄 점검**:
  - Windows 버전 (10 / 11 / Server)
  - 로캘/인코딩 (cp949 vs utf-8)
  - Claude CLI 버전 (호환 가능 범위)
  - Claude 로그인 상태 (OAuth 토큰 유효성)
  - Node.js 설치 여부
  - 한컴 OCX 등록 여부 (HwpCtrl.ocx)
  - 네트워크 (api.anthropic.com 접근 테스트)
  - AV 실시간 보호 상태 (대략 추정)
- 결과를 사용자에게 **체크리스트**로 표시, 문제 있는 항목은 구체적 해결법 (관리자
  cmd 명령 / 설치 링크) 안내.

### D. Graceful Degradation — 완전 실패 방지
**증상**: 어떤 조건 하나라도 나쁘면 앱 전체 사용 불가.

**해결**:
- [향후] 기능별 폴백:
  - SDK 실패 → `-p` subprocess 모드 (느리지만 동작)
  - HWP COM 미등록 → HWP 작업은 차단 + 엑셀/파일시스템 작업은 정상
  - CLI 미설치 → 설정 창으로 유도 + 앱 종료 방지
