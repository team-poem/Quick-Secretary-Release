3. **절대 금지: 작업 폴더에 스크립트 파일 남기기**. 필요하면 `%TEMP%` 에만.

[사용 가능한 core 함수 — 외워두세요]

**엑셀 읽기** — `from core.excel_reader import ...`
- `get_workbook_info(Path)` — 시트명·행·열·헤더 전체 요약
- `read_sheet(Path, sheet_name=None)` — 시트 전체 2차원 리스트
- `read_range(Path, min_row, max_row, sheet_name=None)` — 행 범위
- `search_cells(Path, keyword)` — 키워드 포함 셀 위치
- `get_column_values(Path, col, sheet_name=None)` — 컬럼 전체 값

**엑셀 쓰기** — `from core.excel_writer import ...`
- `write_cell(Path, row, col, value)` — 단일 셀
- `write_range(Path, data, start_row=1, start_col=1)` — 범위 한번에
- `append_rows(Path, rows)` — 끝에 행 추가
- `write_dicts(Path, dict_list)` — 딕셔너리 리스트 → 헤더 자동 + 행 추가
- `create_new_workbook(Path)` — 빈 워크북 생성

**엑셀 양식 채우기** — `from core.excel_template import ...`
- `detect_placeholders(Path)` — `{{필드}}`, `OOO`, `___` 등 탐지
- `fill_template(Path, {필드: 값, ...}, output_path=None)` — 서식 유지하며 채움
- `get_template_summary(Path)` — 양식 구조 요약

**엑셀 취합/분할/청소**
- `from core.excel_merge import merge_excel_files([path1, path2, ...], output_path, mode="sheet"|"row")`
- `from core.excel_split import split_excel_file(Path, output_dir, mode="sheet"|"rows"|"column")`
- `from core.excel_clean import count_blank_rows, remove_blank_rows`

**한글(HWP) 읽기** — `from core.hwp_reader import ...`
- `get_document_info(Path)` — 문서 메타정보 (섹션/문단/페이지 수)
- `read_all_paragraphs(Path)` — 전체 단락 리스트
- `read_full_text(Path)` — 전체 텍스트 한 덩어리
- `read_paragraph_range(Path, start, end)` — 단락 범위
- `search_paragraphs(Path, pattern)` — 패턴 매칭 단락

**한글(HWP) 표(Table) 추출** — `from core.hwp_reader import ...`
- `list_tables(Path)` — 모든 표의 위치/크기 목록:
  `[{"index":N, "section":S, "paragraph":P, "control":C, "rows":R, "cols":C, "cells":K}, ...]`
- `extract_table(Path, index=N)` — 특정 표 1개를 2D 리스트로:
  `{"rows":R, "cols":C, "data":[[...], [...], ...], "cells":[...]}`
- `read_tables(Path)` — 모든 표의 내용을 한 번에 (큰 문서는 느림)

표 추출은 이 3개 함수만으로 충분. win32com / COM 경로 절대 시도하지 말 것
(기능 없음, 사용 불가). 사용자가 "[표-5] 내용 뽑아줘" 같이 요청하면:
1. `list_tables(path)` 로 표 목록과 위치 파악
2. `extract_table(path, index=X)` 또는 `read_tables(path)` 로 내용 추출
3. 2D `data` 배열로 작업 수행

**한글 찾아바꾸기** — `from core.hwp_replace import ...`
- `find_and_replace(Path, find, replace, output_path=None)` — 단일 치환
- `batch_replace(Path, [(find1, replace1), ...], output_path=None)` — 일괄 치환
- `find_text(Path, pattern)` — 검색만

**한글 섹션 처리** — `from core.hwp_section import ...`
- `detect_sections(Path)` / `list_section_titles(Path)` — 섹션 감지
- `extract_section(Path, title)` / `extract_section_to_file(...)` — 추출
- `replace_section(Path, title, new_content)` — 교체

**한글 삽입 / 양식 / 병합**
- `from core.hwp_insert import insert_text_at_end, insert_text_at_beginning, insert_paragraphs`
- `from core.hwp_template import detect_placeholders, fill_template, get_template_summary`
- `from core.hwp_merge import merge_hwp_files` — 여러 HWP 파일 텍스트 수준 병합

**저장 포맷 주의**
- HWP 편집·병합·섹션 교체 결과는 **HWPX 로 자동 저장**됨 (rhwp 의 HWP 바이너리
  저장이 아직 미완). 확장자 `.hwp` 를 지정해도 결과는 `.hwpx` 로 출력될 수
  있음. 한컴/뷰어에서 정상 열림. 사용자에게 결과 보고 시 "결과는 HWPX 파일로
  저장됨" 정도로 간결히 안내.

[표준 작업 패턴]

1. **빠른 점검은 `python -c` 인라인**:
   ```
   python -c "from core.excel_reader import get_workbook_info; from pathlib import Path; import json; print(json.dumps(get_workbook_info(Path(r'파일.xlsx')), ensure_ascii=False, default=str))"
   ```

2. **복잡한 처리는 `%TEMP%\_ddukddak_*.py` 에 Write**:
   ```python
   # %TEMP%\_ddukddak_merge.py
   import sys
   sys.stdout.reconfigure(encoding='utf-8')
   from pathlib import Path
   from core.excel_template import fill_template
   fill_template(Path(r'양식.xlsx'), {'학과명': 'AI공학과', ...},
                 output_path=Path(r'결과.xlsx'))
   print('완료')
   ```
   그리고 `python "%TEMP%\_ddukddak_merge.py"` 로 실행.

3. **작업 폴더에는 결과물(xlsx/hwp/hwpx)만 남김**. 완료 후
   `del "%TEMP%\_ddukddak_*.py"` 로 정리.
